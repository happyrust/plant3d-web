# 前端状态持久化方案

## 1. 背景
用户在刷新页面后，当前的前端状态（如模型树的展开状态、三维视图中已加载的模型、Dock 面板中的表单数据等）会丢失，导致工作上下文中断。为了提升用户体验，需要实现前端状态的持久化与重载。

## 2. 状态范围
本方案主要覆盖以下三类状态的持久化：

1.  **模型树状态**：
    *   节点展开状态 (`expandedIds`)
    *   (可选) 选中节点 (`selectedIds`)
2.  **三维视图状态**：
    *   已加载并显示的模型节点 (RefNos)
    *   (可选) 相机视角 (Camera Position/Target)
3.  **Dock 页面数据**：
    *   各功能面板（如水力计算、属性面板等）中的用户输入数据

## 3. 技术方案设计

核心采用 `localStorage` 配合 Vue Composable (`useStorage` 或自定义封装) 来实现响应式数据的持久化。

### 3.1 通用基础设施：`useLocalStorageRef`

创建一个通用的 Composable，用于将 `ref` 与 `localStorage` 双向绑定。

```typescript
// src/composables/useLocalStorage.ts (新建)
import { ref, watch, type Ref } from 'vue';

export function useLocalStorageRef<T>(key: string, defaultValue: T): Ref<T> {
  // 1. 尝试从 localStorage 读取
  const stored = localStorage.getItem(key);
  let initialValue: T = defaultValue;

  if (stored) {
    try {
      initialValue = JSON.parse(stored);
    } catch (e) {
      console.warn(`[useLocalStorageRef] Failed to parse key "${key}", using default.`);
    }
  }

  const data = ref<T>(initialValue) as Ref<T>;

  // 2. 监听变化并写入 localStorage
  watch(
    data,
    (newValue) => {
      if (newValue === undefined || newValue === null) {
        localStorage.removeItem(key);
      } else {
        localStorage.setItem(key, JSON.stringify(newValue));
      }
    },
    { deep: true } 
  );

  return data;
}

// 针对 Set类型的特殊封装 (因为 Set 无法直接 JSON 序列化)
export function useLocalStorageSet<T>(key: string): Ref<Set<T>> {
    const listRef = useLocalStorageRef<T[]>(key, []);
    const setRef = ref(new Set(listRef.value)) as Ref<Set<T>>;

    watch(setRef, (newSet) => {
        listRef.value = Array.from(newSet);
    }, { deep: true });

    return setRef;
}
```

### 3.2 模型树状态持久化

**修改目标**：`src/composables/usePdmsOwnerTree.ts` 和 `src/composables/useRoomTree.ts`

**实现逻辑**：
将内部的 `expandedIds` 从普通 `ref` 替换为持久化 `ref`。

```typescript
// 以 usePdmsOwnerTree.ts 为例
// 替换: const expandedIds = ref<Set<string>>(new Set());
const expandedIds = useLocalStorageSet<string>('plant3d:tree:pdms:expanded');
```

*注意：需要处理 key 的命名空间，区分 PDMS 树和 Room 树。*

### 3.3 三维视图模型重载

**问题**：Viewer 中的模型是按需加载的（通过 `showModelByRefno` 或 lazy loading）。刷新页面后，Viewer 是空的，必须重新触发加载流程。

**实现逻辑**：
1.  **记录已加载模型**：
    在 `useModelGeneration` 或 `ModelTreePanel` 中，维护一个 `loadedRefnos` 的持久化列表。
    每当调用 `showModelByRefno` 成功，或通过 Checkbox 显示了一个 RefNo (且触发了加载)，将该 RefNo 加入列表。
2.  **初始化重载**：
    在 `App.vue` 或 `ViewerPanel.vue` 初始化阶段（Viewer 准备就绪后），读取 `loadedRefnos` 列表，自动遍历并调用 `showModelByRefno` (或其他批量加载接口) 恢复场景。

**数据结构**：
Key: `plant3d:viewer:loaded-models`
Value: `["17496_266204", "123_456", ...]`

**代码切入点**：
*   在 `src/composables/useModelGeneration.ts` 中添加逻辑，或者新建 `src/composables/useViewerPersistence.ts` 专门管理上下文恢复。

### 3.4 Dock 页面数据持久化

**问题**：目前 Panel 组件（如 `HydraulicPanel.vue`）的状态是组件级局部状态，销毁即逝。

**实现逻辑**：
在各具体的 Panel 组件中，使用 `useLocalStorageRef` 替换原本的 `ref` 初始化。

**示例 (HydraulicPanel.vue)**：

```typescript
// 原代码
// const airSpeedDuct = ref(5.0);

// 修改为
const airSpeedDuct = useLocalStorageRef('plant3d:panel:hydraulic:airSpeedDuct', 5.0);
const ductLength = useLocalStorageRef('plant3d:panel:hydraulic:ductLength', 10.0);
// ... 其他字段
```

建议为每个 Panel 定义一个统一的 State 对象进行存储，避免 Key 过于零散：

```typescript
const hydraulicState = useLocalStorageRef('plant3d:panel:hydraulic', {
  airSpeedDuct: 5.0,
  ductLength: 10.0,
  // ...
});
// 模板中使用 hydraulicState.value.airSpeedDuct
```

## 4. 实施步骤

1.  **创建基础工具**：
    *   新建 `src/composables/useLocalStorage.ts`。
2.  **模型树改造**：
    *   修改 `usePdmsOwnerTree.ts`，持久化 `expandedIds`。
    *   修改 `useRoomTree.ts`，持久化 `expandedIds`。
3.  **视图恢复机制**：
    *   新建 `useViewerPersistence.ts`。
    *   在 `ModelTreePanel` 加载模型成功时记录 RefNo。
    *   在 `ViewerPanel` `onMounted` 且 Viewer 初始化完成后，调用恢复逻辑。
4.  **Dock 面板改造**：
    *   逐个挑选重要的 Panel（如水力计算），将其关键状态字段替换为持久化存储。

## 5. 风险与注意事项

*   **数据过时**：LocalStorage 中的数据可能与后端最新模型不一致。
    *   *对策*：对于 RefNo 列表，如果加载失败（后端已删除该模型），应通过 `try/catch` 捕获并从列表中移除。
*   **性能问题**：如果 `expandedIds` 非常多（数千个），JSON 序列化可能造成轻微卡顿。
    *   *对策*：一般用户手动展开的数量有限，风险可控。如有必要，可添加防抖 (Debounce) 写入。
*   **版本兼容性**：代码更新后，旧的 LocalStorage 数据结构可能导致报错。
    *   *对策*：在 `useLocalStorageRef` 中通过 `try/catch` 解析，如果结构不匹配或报错，降级为默认值。建议 Key 加上版本后缀，如 `v1`。
