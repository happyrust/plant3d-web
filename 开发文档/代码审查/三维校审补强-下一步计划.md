## 下一步任务

### H1: 后端联调验证（启动后端 + HTTP 请求验证）

**目的**：确认新增 API 在实际运行环境下可用。

步骤：
1. 启动后端：`WEB_SERVER_PORT=3910 cargo run --bin web_server --features web_server`
2. 创建测试 token 和任务
3. 逐一验证：
   - `POST /api/review/annotation-states/apply` — 写入状态
   - `GET /api/review/annotation-states?form_id=...&task_id=...` — 查询状态
   - `POST /api/review/comments` — 带 formId/taskId 创建评论
   - `GET /api/review/comments/by-annotation/{id}?form_id=...` — 上下文过滤
   - `DELETE /api/review/comments/item/{id}` — 非作者应 403
   - `PATCH /api/review/comments/item/{id}` — 编辑评论

### H2: HTTP 合同脚本

**位置**：`scripts/review-annotation-flow-contract.ts`

覆盖场景：
1. 创建 form_id + 任务
2. 保存带批注的确认记录（触发 A3 同步）
3. 评论写入（带 formId/taskId）
4. 批注状态写入（fixed → pending review）
5. `workflow/verify active` 应阻塞未处理批注
6. 设计侧全部标记已修改后，verify 通过
7. 校核侧待确认时阻塞
8. 校核侧同意后通过
9. 两个 form_id 同 annotation_id 评论不串
10. 非作者删除评论返回 403
11. 清理测试数据
