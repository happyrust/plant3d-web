# 模型加载策略升级方案：DuckDB WASM 虚拟化读取

## 1. 背景与目标

当前模型加载采用 "按次生成 JSON Bundle" 的模式（`manifest.json` + `instances/*.json`）。
**存在问题**：
1. **重复计算**：每次请求都重新生成数据，无法有效利用已有的生成结果。
2. **缓存困难**：Bundle 是临时的，难以在多次会话间复用。
3. **并发限制**：生成过程阻塞，无法支持流式持续生成和加载。

**升级目标**：
1. **持久化存储**：按数据库编号（`dbno`）将模型数据保存为 DuckDB 数据库文件。
2. **虚拟化读取**：浏览器端不下载整个文件，而是通过 HTTP Range Requests 按需读取。
3. **增量生成**：每次生成时检测已存在的数据，仅生成新增部分。

---

## 2. 核心理念：虚拟化读取 (HTTP Range Requests)

### 2.1 什么是虚拟化读取？

传统方式是将整个数据库文件下载到浏览器内存，而**虚拟化读取**的核心是：
> 把远程文件当成一块"虚拟硬盘"，DuckDB-WASM 只读取它需要的字节。

DuckDB-WASM 支持通过 HTTP Range 头只读取它需要的字节：
1. **初始化阶段**：DuckDB 只读取文件的 Header 和 Footer（几 KB）
2. **查询阶段**：根据 SQL 查询内容，自动向服务器请求需要的"页（Pages）"
3. **极致效率**：一个 100MB 的数据库，查询某条记录可能只传输 100KB

### 2.2 关键配置要求

| 配置项 | 说明 |
| :--- | :--- |
| 服务器响应头 | 必须返回 `Accept-Ranges: bytes` |
| 支持的请求头 | 必须正确处理 `Range: bytes=1000-2000` |
| 推荐服务器 | Nginx / Minio / S3 / 任何支持 Range 的静态文件服务 |

## 3. 后端开发方案 (gen-model-fork)

### 3.1 技术选型
*   **数据库引擎**: 使用 `duckdb` Rust crate 创建和写入 `.duckdb` 文件。
*   **存储结构**: 基于文件系统的分区存储。

### 3.2 存储目录结构
```text
output/
  └── database_models/
      └── {dbno}/              # 按数据库编号分区
          ├── meta.json        # 元数据（版本、最后更新时间、记录数等）
          └── model.duckdb     # DuckDB 数据库文件
```

### 3.3 数据 Schema (DuckDB Tables)

> **设计原则**: aabb 和 trans 表使用 hash 作为主键，与 SurrealDB 定义保持一致。相同的变换矩阵或包围盒只存储一份，多个实例可共享。

#### 3.3.1 变换矩阵表 (trans)

与 SurrealDB 定义保持一致：

```sql
CREATE TABLE trans (
    hash        VARCHAR PRIMARY KEY,    -- 变换矩阵的哈希值 (与 SurrealDB trans:⟨hash⟩ 一致)
    rotation    VARCHAR,                -- 四元数 [x, y, z, w] (JSON)
    scale       VARCHAR,                -- 缩放 [x, y, z] (JSON)
    translation VARCHAR                 -- 平移 [x, y, z] (JSON)
);
```

#### 3.3.2 包围盒表 (aabb) - 使用 R-Tree 空间索引

> **注意**: 需要加载 `spatial` 扩展。采用 2D R-Tree + Z 范围过滤方案（DuckDB spatial 以 2D GIS 为主）。

```sql
-- 加载 spatial 扩展
INSTALL spatial;
LOAD spatial;

CREATE TABLE aabb (
    hash    VARCHAR PRIMARY KEY,       -- 包围盒的哈希值 (与 SurrealDB aabb:⟨hash⟩ 一致)
    bbox    GEOMETRY,                  -- XY 平面包围盒: ST_MakeEnvelope(min_x, min_y, max_x, max_y)
    min_z   DOUBLE,                    -- 最小 Z
    max_z   DOUBLE                     -- 最大 Z
);

-- R-Tree 空间索引（用于 XY 平面快速查询）
CREATE INDEX idx_aabb_rtree ON aabb USING RTREE (bbox);

-- Z 范围普通索引（配合 R-Tree 使用）
CREATE INDEX idx_aabb_z ON aabb(min_z, max_z);
```

#### 3.3.3 实例表 (instance)

```sql
CREATE TABLE instance (
    refno           VARCHAR PRIMARY KEY,    -- 唯一标识符 '24383_73962'
    noun            VARCHAR,                -- 节点类型 'BRAN', 'EQUI', 'STRT', 'DAMP' 等
    name            VARCHAR,                -- 元素名称
    trans_hash      VARCHAR,                -- 引用 trans 表的 hash
    aabb_hash       VARCHAR,                -- 引用 aabb 表的 hash
    color_index     INTEGER,                -- 颜色索引
    lod_mask        INTEGER,                -- LOD 掩码
    spec_value      INTEGER,                -- 规格值
    properties      VARCHAR                 -- 额外的属性数据 (JSON 格式)
);

CREATE INDEX idx_noun ON instance(noun);
CREATE INDEX idx_trans_hash ON instance(trans_hash);
CREATE INDEX idx_aabb_hash ON instance(aabb_hash);
```

#### 3.3.4 几何体表 (geo)

每个实例可以包含多个几何体（一对多关系）：

```sql
CREATE TABLE geo (
    id            VARCHAR PRIMARY KEY,    -- 复合主键: refno_number (如 '12345_67890_0')
    refno         VARCHAR NOT NULL,       -- 关联的实例 refno
    geo_hash      VARCHAR NOT NULL,       -- 几何体哈希 (指向 .glb 文件)
    geo_transform VARCHAR                 -- 几何体级别的局部变换矩阵 (JSON 格式)
);

CREATE INDEX idx_geo_refno ON geo(refno);
CREATE INDEX idx_geo_geo_hash ON geo(geo_hash);
```

> **ID 格式**: `{refno}_{number}`，其中 `number` 是该实例下几何体的序号（从 0 开始）。例如 `24381_46952_0`、`24381_46952_1`。

#### 3.3.5 数据关系说明

```text
trans (hash) <────── instance.trans_hash
                            │
aabb (hash)  <────── instance.aabb_hash
                            │
                       instance (refno)
                            │
                            └────> geo (N条, 每条有 geo_hash + geo_transform)

最终渲染矩阵 = trans.d × geo.geo_transform
```

### 3.4 数据来源与同步

| 字段 | 数据来源 | 说明 |
| :--- | :--- | :--- |
| `trans.hash` | SurrealDB `trans:⟨hash⟩` | 变换矩阵的哈希值，直接同步 |
| `trans.d` | SurrealDB `trans.d` | 16 位变换矩阵数据 |
| `aabb.hash` | SurrealDB `aabb:⟨hash⟩` | 包围盒的哈希值，直接同步 |
| `aabb.bbox` | SurrealDB `aabb.d` | XY 坐标转为 `ST_MakeEnvelope` |
| `aabb.min_z/max_z` | SurrealDB `aabb.d` | Z 轴坐标 |
| `geo_hash` | 几何生成器 | 单位几何体的哈希标识 |
| `geo_transform` | 几何生成器 | 几何体相对于实例原点的局部变换 |

> **同步策略**: 生成模型时，从 SurrealDB 直接同步 `trans` 和 `aabb` 表数据（保持 hash 主键一致），instances 表仅存储 `trans_hash` 和 `aabb_hash` 引用。

### 3.5 生成流程改造

1.  **接收请求**: 收到生成请求 (包含 `refnos`)。
2.  **存在性检查**:
    *   打开目标 `dbno` 下的 `model.duckdb`。
    *   查询已存在的 `refnos`：`SELECT refno FROM instances WHERE refno IN (...)`。
    *   筛选出**未存在**的 `refnos` 进行生成。
3.  **增量生成**:
    *   **Trans/AABB**: 对所有涉及的变换矩阵和包围盒，执行 `INSERT OR IGNORE INTO trans/aabb`。注意 `aabb` 需转换为 `ST_MakeEnvelope`。
    *   **Geos**: 对缺失的 refnos 执行几何生成逻辑。
4.  **数据写入**:
    *   使用 `INSERT OR REPLACE INTO instances ...` 写入实例数据（关联 `trans_hash`, `aabb_hash`）。
    *   更新 `meta.json` 的元数据。

### 3.6 服务器配置 (Nginx)

```nginx
location /files/database_models/ {
    alias /path/to/output/database_models/;
    
    # 启用 Range Requests
    add_header Accept-Ranges bytes;
    
    # 允许 CORS (如果需要跨域)
    add_header Access-Control-Allow-Origin *;
    add_header Access-Control-Expose-Headers Content-Length,Content-Range;
    
    # 长效缓存 (数据库文件变化时应更新文件名或查询参数)
    add_header Cache-Control "public, max-age=31536000";
}
```

### 3.7 API 变更

*   `POST /api/model/generate`: 仅触发生成/更新，返回任务状态。
*   `GET /api/model/{dbno}/info`: 返回该数据库的元数据（记录数、最后更新时间等）。
*   `GET /files/database_models/{dbno}/model.duckdb`: 静态文件服务 (支持 Range Requests)。

## 4. 前端开发方案 (plant3d-web)

### 4.1 技术选型
*   **读取库**: `@duckdb/duckdb-wasm` —— 支持 HTTP Range Requests 的虚拟化读取。
*   **优势**: 无需下载整个文件，按需请求数据页，适合大规模模型数据。

### 4.2 DuckDB 初始化

```typescript
// utils/duckdb.ts
import * as duckdb from '@duckdb/duckdb-wasm';
import duckdb_wasm from '@duckdb/duckdb-wasm/dist/duckdb-mvp.wasm?url';
import duckdb_worker from '@duckdb/duckdb-wasm/dist/duckdb-browser-mvp.worker.js?url';

let db: duckdb.AsyncDuckDB | null = null;

export async function initDuckDB(): Promise<duckdb.AsyncDuckDB> {
  if (db) return db;
  
  const MANUAL_BUNDLES: duckdb.DuckDBBundles = {
    mvp: {
      mainModule: duckdb_wasm,
      mainWorker: duckdb_worker,
    },
  };
  
  const bundle = await duckdb.selectBundle(MANUAL_BUNDLES);
  const worker = new Worker(bundle.mainWorker!);
  const logger = new duckdb.ConsoleLogger();
  
  db = new duckdb.AsyncDuckDB(logger, worker);
  await db.instantiate(bundle.mainModule);
  
  return db;
}
```

### 4.3 核心加载逻辑

```typescript
// composables/useModelDatabase.ts
import { initDuckDB } from '@/utils/duckdb';
import * as duckdb from '@duckdb/duckdb-wasm';

export function useModelDatabase() {
  let conn: duckdb.AsyncDuckDBConnection | null = null;

  /**
   * 注册远程数据库 URL（不会下载整个文件）
   */
  async function registerRemoteDatabase(dbno: string): Promise<void> {
    const db = await initDuckDB();
    
    const url = `/files/database_models/${dbno}/model.duckdb`;
    
    // 注册远程 URL，DuckDB 只会先读 Header 和 Footer (几 KB)
    await db.registerFileURL(
      'model.db',
      url,
      duckdb.DuckDBDataProtocol.HTTP,
      true  // 只读模式
    );
    
    conn = await db.connect();
  }

  /**
   * 查询所有实例基本信息（分页）
   */
  async function queryInstances(page: number = 0, pageSize: number = 1000) {
    if (!conn) throw new Error('Database not connected');
    
    const offset = page * pageSize;
    const result = await conn.query(`
      SELECT refno, noun, name, refno_transform, color_index, lod_mask, spec_value
      FROM model.db.instances
      LIMIT ${pageSize} OFFSET ${offset}
    `);
    
    return result.toArray();
  }

  /**
   * 查询单个实例的完整数据（包含所有 geos 和变换矩阵）
   */
  async function queryInstanceWithGeos(refno: string) {
    if (!conn) throw new Error('Database not connected');
    
    // 查询实例基本信息，JOIN trans 和 aabb 表获取完整数据
    const instResult = await conn.query(`
      SELECT 
        i.*,
        t.d AS world_transform,
        ST_XMin(a.bbox) as min_x, ST_YMin(a.bbox) as min_y, 
        ST_XMax(a.bbox) as max_x, ST_YMax(a.bbox) as max_y, 
        a.min_z, a.max_z
      FROM model.db.instances i
      LEFT JOIN model.db.trans t ON i.trans_hash = t.hash
      LEFT JOIN model.db.aabb a ON i.aabb_hash = a.hash
      WHERE i.refno = '${refno}'
    `);
    const instance = instResult.toArray()[0];
    if (!instance) return null;
    
    // 查询该实例的所有几何体
    const geosResult = await conn.query(`
      SELECT geo_hash, geo_index, geo_transform 
      FROM model.db.geos 
      WHERE refno = '${refno}' 
      ORDER BY geo_index
    `);
    
    return {
      ...instance,
      geos: geosResult.toArray(),
    };
  }

  /**
   * 按 geo_hash 分组查询（用于实例化渲染）
   * 返回使用该几何体的所有实例及其变换矩阵
   */
  async function queryByGeoHash(geoHash: string) {
    if (!conn) throw new Error('Database not connected');
    
    // JOIN 查询：获取使用该 geo_hash 的所有实例和对应的变换矩阵
    const result = await conn.query(`
      SELECT 
        i.refno,
        t.d AS world_transform,
        g.geo_transform,
        i.color_index
      FROM model.db.geos g
      JOIN model.db.instances i ON g.refno = i.refno
      JOIN model.db.trans t ON i.trans_hash = t.hash
      WHERE g.geo_hash = '${geoHash}'
    `);
    
    return result.toArray();
  }

  /**
   * 获取所有唯一的 geo_hash 列表
   */
  async function getUniqueGeoHashes(): Promise<string[]> {
    if (!conn) throw new Error('Database not connected');
    
    const result = await conn.query(`
      SELECT DISTINCT geo_hash FROM model.db.geos
    `);
    
    return result.toArray().map(row => row.geo_hash);
  }

  /**
   * 视锥体剔除查询：获取在指定范围内的实例
   */
  async function queryByBoundingBox(
    minX: number, maxX: number,
    minY: number, maxY: number,
    minZ: number, maxZ: number
  ) {
    if (!conn) throw new Error('Database not connected');
    
    // 使用 R-Tree 空间索引进行高效查询
    // ST_Intersects 利用 R-Tree 索引加速 XY 平面查询，Z 范围单独过滤
    const result = await conn.query(`
      SELECT 
        i.refno, 
        ST_XMin(a.bbox) as min_x, ST_YMin(a.bbox) as min_y, 
        ST_XMax(a.bbox) as max_x, ST_YMax(a.bbox) as max_y,
        a.min_z, a.max_z
      FROM model.db.instances i
      JOIN model.db.aabb a ON i.aabb_hash = a.hash
      WHERE ST_Intersects(a.bbox, ST_MakeEnvelope(${minX}, ${minY}, ${maxX}, ${maxY}))
        AND a.max_z >= ${minZ} AND a.min_z <= ${maxZ}
    `);
    
    return result.toArray();
  }

  async function close() {
    if (conn) {
      await conn.close();
      conn = null;
    }
  }

  return {
    registerRemoteDatabase,
    queryInstances,
    queryInstanceWithGeos,
    queryByGeoHash,
    getUniqueGeoHashes,
    queryByBoundingBox,
    close,
  };
}
```

### 4.4 渲染流程改造

```typescript
// 加载流程示意
async function loadModel(dbno: string) {
  const modelDb = useModelDatabase();
  
  // 1. 注册远程数据库 (只读取 Header，极快)
  await modelDb.registerRemoteDatabase(dbno);
  
  // 2. 获取所有唯一的 geo_hash
  const geoHashes = await modelDb.getUniqueGeoHashes();
  
  // 3. 对每个 geo_hash，加载 GLB 并创建 InstancedMesh
  for (const geoHash of geoHashes) {
    // 3.1 加载 GLB 几何体
    const geometry = await loadGLB(`/files/geos/${geoHash}.glb`);
    
    // 3.2 查询该 geo_hash 的所有实例 (DuckDB 按需请求数据)
    const instances = await modelDb.queryByGeoHash(geoHash);
    
    // 3.3 创建 InstancedMesh
    const mesh = new THREE.InstancedMesh(geometry, material, instances.length);
    
    instances.forEach((inst, i) => {
      // 计算最终矩阵: world_transform × geo_transform
      const worldMat = new THREE.Matrix4().fromArray(inst.world_transform);
      const geoMat = new THREE.Matrix4().fromArray(inst.geo_transform);
      const finalMat = worldMat.multiply(geoMat);
      mesh.setMatrixAt(i, finalMat);
    });
    
    scene.add(mesh);
  }
}
```

### 4.5 性能对比

| 指标 | 传统下载方案 | 虚拟化读取方案 |
| :--- | :--- | :--- |
| 初始加载时间 | 下载整个文件 (100MB+) | 仅读取 Header (~10KB) |
| 内存占用 | 整个数据库驻留内存 | 按需加载，自动释放 |
| 查询延迟 | 本地查询，极快 | 网络往返 + 页读取 |
| 适用场景 | 数据量小，全量使用 | 数据量大，按需使用 |
