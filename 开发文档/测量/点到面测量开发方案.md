# 点到面测量开发方案

## 1. 需求概述

实现“点到面”（Point to Mesh）的测量功能，允许用户计算模型表面任意一点到另一个对象（Mesh）几何表面的最短距离。
该功能需要集成到 Ribbon 菜单中，并在激活时提供浮动向导窗口，引导用户完成操作。测量结果应以 CAD 风格（带箭头直线 + 文字标签）展示。

## 2. 功能设计

### 2.1 入口设计
*   **Ribbon 菜单**：在 `测量` (Measurement) 选项卡的 `创建` (Create) 组中添加 `点到面` 按钮。
    *   图标：使用 `ruler-combined` 或类似图标。
    *   标签：“点到面”。

### 2.2 交互流程
1.  **激活**：用户点击 Ribbon 按钮，进入 `measure_point_to_object` 模式。
2.  **向导浮窗**：屏幕出现浮动窗口（Wizard），提示当前步骤。
    *   **步骤 1**：“请点击模型表面选择起始点”。
        *   用户点击模型表面 -> 记录起始点坐标 -> 在场景中绘制临时标记（如小球）-> 进入步骤 2。
    *   **步骤 2**：“请点击选择目标对象”。
        *   用户点击目标对象（如墙体、管道） -> 系统计算起始点到该对象几何面的最短距离。
3.  **结果展示**：
    *   计算成功后，在场景中绘制测量线。
    *   测量线样式：CAD 风格，包含线段、两端箭头、距离数值标签。
    *   退出测量模式或保留模式以便进行下一次测量（参考 CAD 习惯，通常保留）。

### 2.3 数据结构
复用现有的 `DistanceMeasurementRecord`，但可能需要扩展字段或使用特定的 `subKind` 来标识这是“点到面”的测量，以便区分渲染样式（如果默认距离测量样式不满足 CAD 箭头要求）。

## 3. 技术实现方案

### 3.1 核心逻辑 (`useXeokitTools.ts`)
*   **状态管理**：
    *   `toolMode`: `measure_point_to_object`.
    *   `pointToRefnoStart`: 存储起始点信息 `{ entityId, worldPos }`.
*   **计算逻辑**：
    *   复用已实现的 `getClosestPointOnEntity(entity, point)` 函数。
    *   该函数遍历目标 Entity 的几何数据（Vertices/Indices），计算点到三角形的最短距离。

### 3.2 UI 组件
*   **Ribbon 配置** (`ribbonConfig.ts`)：配置新按钮。
*   **浮动向导** (`MeasurementWizard.vue`)：
    *   这是一个新的 Vue 组件，悬浮在 3D 视口之上。
    *   监听 `store.toolMode` 和 `useXeokitTools` 的状态（如 `pointToRefnoStart` 是否有值）来切换提示文案。
    *   提供“取消”按钮以退出模式。

### 3.3 渲染实现 (`CAD Style`)
*   **当前插件**：`DistanceMeasurementsPlugin` 默认样式可能仅为简单的线段和标签，不一定带 CAD 风格箭头。
*   **定制方案**：
    *   **方案 A (插件配置)**：检查 `xeokit` 插件是否支持配置箭头端点。
    *   **方案 B (自定义绘制)**：如果不使用插件，改用自定义的 `LineSet` (画线) + `Mesh` (画箭头 Cone) + `Annotation` (画文字标签)。
    *   **推荐方案**：为了保证“CAD 标准”，建议采用 **方案 B** 或 扩展插件。我们将创建一个新的辅助函数 `createCadMeasurement(...)` 来负责创建这些组合图形。

## 4. 开发计划

### 第一阶段：UI 集成 (UI Integration)
1.  修改 `src/ribbon/ribbonConfig.ts`，添加“点到面”按钮。
2.  创建 `src/components/tools/MeasurementWizard.vue` 组件，实现浮动窗口逻辑。
3.  在 `App.vue` 或 `AiosPrepackViewer.vue` 中挂载该 Wizard 组件。

### 第二阶段：交互优化 (Interaction Refinement)
1.  完善 `useXeokitTools.ts` 中的点击处理。
2.  添加起始点视觉反馈（点击第一下时，在点击处生成一个临时的小圆点 `Mesh`）。
3.  确保状态机（Start -> End -> Reset）流转顺畅。

### 第三阶段：渲染实现 (Rendering)
1.  实现 `drawCadMeasurement(start, end, distance)` 函数。
    *   使用 `LineSet` 绘制线段。
    *   计算向量方向，在两端生成 `Mesh` (Cone 几何体) 作为箭头，并正确旋转朝向。
    *   在其中点生成 `Annotation` 或 `Label` 显示距离数值。
2.  将计算结果传递给该渲染函数进行绘制。

### 第四阶段：验证与测试
1.  测试不同几何体（管道、长方体、不规则 Mesh）的距离计算准确性。
2.  测试交互流程的易用性（取消、连续测量）。

## 5. 预期效果
用户点击按钮后，跟随屏幕指引，两步点击即可获得精准的空间距离数据，且视觉风格专业、清晰。
