#### 必须拍板的 6 项

**1. ReviewPanel 的正式分区结构**
- [ ] 确认分区方案：任务上下文区、流转区、确认记录区、辅助数据区、同步区
- [ ] 确认是否保留附件区和 embed reviewer 提示
- [ ] 确认各分区的展开/收起策略

**2. 当前任务上下文的正式字段口径**
- [ ] 确认正式字段：`taskId/formId/checker/approver/currentNode/workflowHistory/returnReason`
- [ ] 确认 `formId` 缺失时是否允许为空（不回落 `task.id`）
- [ ] 确认 `reviewerName` 仅作为兼容兜底，不作主渲染字段

**3. 流转入口统一走 /submit、/return**
- [ ] 确认 `/submit` 为唯一正向推进接口
- [ ] 确认 `/return` 为唯一逆向退回接口
- [ ] 确认是否仍需 `/start-review` 前置
- [ ] 确认批准节点是否单独走 `/approve`

**4. 确认记录、流转历史、评论三条数据线严格分治**
- [ ] 确认记录接口只承载确认快照
- [ ] 确认不与 workflow history 混写
- [ ] 确认不与评论混写
- [ ] 确认查询、删除、清空的权限

**5. ReviewAuxData 的正式入参来源**
- [ ] 确认 `project_id/model_refnos/major/requester_id/form_id` 的必填性
- [ ] 确认 `formId` 正式来自任务上下文，不回落 `task.id`
- [ ] 确认短期是否继续沿用 `UCode/UKey`

**6. 最小实时刷新事件集和用户通道语义**
- [ ] 确认 M4 需要的最小 WS 事件集
- [ ] 确认至少覆盖任务变更、确认记录保存
- [ ] 确认 WS 用户通道绑定当前登录用户还是 `reviewerId`
