# 校审系统优化实施文档

## 概述

本文档记录了 Plant3D-Web 校审系统的优化实施过程，将数据从 localStorage 迁移到后端 SurrealDB 数据库。

## 变更清单

### 1. 新增文件

| 文件 | 说明 |
|------|------|
| `src/api/reviewApi.ts` | 校审 API 调用模块，包含提资单、确认记录、评论、附件等 API |
| `开发文档/三维校审/校审流程分析.md` | 校审流程分析文档 |
| `开发文档/三维校审/数据库设计.md` | SurrealDB 数据库表结构设计 |
| `开发文档/三维校审/优化实施文档.md` | 本文档 |

### 2. 修改文件

| 文件 | 变更说明 |
|------|----------|
| `src/composables/useReviewStore.ts` | 重构为支持后端 API，添加 WebSocket 实时通知 |
| `src/composables/useUserStore.ts` | 重构为支持后端 API，添加任务管理和 WebSocket |

## 实现的功能

### 后端集成

1. **提资单管理 API**
   - 创建/查询/更新/删除提资单
   - 审核操作（开始审核/通过/驳回/取消）
   - 审核历史查询

2. **确认记录 API**
   - 保存批注和测量数据快照
   - 按任务查询确认记录
   - 删除/清空确认记录

3. **评论系统 API**
   - 添加批注评论
   - 查询批注评论
   - 删除评论

4. **用户管理 API**
   - 获取用户列表
   - 获取当前用户
   - 获取可用审核人员

### WebSocket 实时通知

1. **全局通知通道** (`/ws/review`)
   - 任务创建/更新/提交/通过/驳回/取消通知

2. **用户专属通道** (`/ws/review/user/{userId}`)
   - 确认记录保存通知
   - 任务状态变更通知
   - 新评论通知

3. **自动重连机制**
   - 最多 5 次重连尝试
   - 3 秒重连延迟

### 向后兼容

1. **本地模式支持**
   - 可通过 `setUseBackend(false)` 切换到本地模式
   - 本地模式使用 mockUsers 和 localStorage

2. **数据版本迁移**
   - 自动从 V1 格式迁移到 V2 格式
   - 兼容旧版 localStorage 数据

## API 端点总览

### 提资单

```
POST   /api/review/tasks              - 创建提资单
GET    /api/review/tasks              - 获取任务列表
GET    /api/review/tasks/:id          - 获取任务详情
PATCH  /api/review/tasks/:id          - 更新任务
DELETE /api/review/tasks/:id          - 删除任务
POST   /api/review/tasks/:id/start-review  - 开始审核
POST   /api/review/tasks/:id/approve  - 通过审核
POST   /api/review/tasks/:id/reject   - 驳回审核
POST   /api/review/tasks/:id/cancel   - 取消任务
GET    /api/review/tasks/:id/history  - 获取审核历史
```

### 确认记录

```
POST   /api/review/records            - 保存确认记录
GET    /api/review/records/:taskId    - 获取任务记录
DELETE /api/review/records/:recordId  - 删除记录
DELETE /api/review/records/task/:taskId  - 清空任务记录
```

### 评论

```
POST   /api/review/comments           - 添加评论
GET    /api/review/comments/:annotationId  - 获取评论
DELETE /api/review/comments/:commentId  - 删除评论
```

### 用户

```
GET    /api/users                     - 获取用户列表
GET    /api/users/me                  - 获取当前用户
GET    /api/users/reviewers           - 获取审核人员
```

### WebSocket

```
ws://host/ws/review                   - 全局审核通知
ws://host/ws/review/user/:userId      - 用户专属通知
```

## 后端实现要求

后端需要实现上述 API 端点，数据存储到 SurrealDB。详细的数据库表结构设计见 `数据库设计.md`。

### 核心表

1. `users` - 用户表
2. `review_tasks` - 提资单/审核任务表
3. `review_records` - 审核确认记录表
4. `review_history` - 审核历史日志表
5. `review_comments` - 批注评论表
6. `review_attachments` - 附件表

## 使用说明

### 启用后端模式（默认）

```typescript
import { useUserStore } from '@/composables/useUserStore';

const userStore = useUserStore();

// 初始化（加载用户和任务数据，连接 WebSocket）
await userStore.initialize();
```

### 切换到本地模式

```typescript
import { useUserStore } from '@/composables/useUserStore';
import { useReviewStore } from '@/composables/useReviewStore';

const userStore = useUserStore();
const reviewStore = useReviewStore();

// 切换到本地模式
userStore.setUseBackend(false);
reviewStore.setUseBackend(false);
```

## 测试建议

1. **后端 API 不可用时**
   - 系统应自动回退到本地模式
   - 使用 mockUsers 数据

2. **WebSocket 连接失败**
   - 自动重连机制应正常工作
   - 超过重连次数后显示错误信息

3. **数据迁移**
   - 旧版 localStorage 数据应自动迁移
