# 三维校审 M5 批注工具与确认动作会议 Checklist

> 用途：开会快速对齐 + 会后直接派单  
> 主设计源：`ui/三维校审/review-reviewer.pen`  
> 总览基线：`ui/三维校审/review-flow.pen`（仅用于跨角色流程与工作台衔接对照）  
> 对应文档：`开发文档/三维校审/三维校审M5批注工具与确认动作任务单.md`

## 一、会议目标

- [ ] 对齐 M5 只做“批注工具与确认动作”，不扩到 M6 / M7
- [ ] 拍板工具栏、批注列表、确认动作三块正式形态
- [ ] 明确旧 `AnnotationPanel` 的迁移与过渡方案
- [ ] 会后可以直接按 owner 派单开工

## 二、会前必读

- [ ] 阅读 `开发文档/三维校审/三维校审M5批注工具与确认动作任务单.md`
- [ ] 阅读 `开发文档/三维校审/三维校审重构执行方案.md`
- [ ] 阅读 `开发文档/三维校审/三维校审M0页面接口字段映射表.md`
- [ ] 对照 `ui/三维校审/review-reviewer.pen` 确认批注工具、列表与确认动作主设计
- [ ] 如需核对与工作台和跨角色流程的衔接，再参考 `ui/三维校审/review-flow.pen`
- [ ] 前端确认范围落点：`AnnotationPanel` / `ReviewConfirmation` / `ViewerPanel` / `useToolStore`
- [ ] 图形与后端确认 annotation 数据结构与确认记录边界

- [ ] 工具栏是否正式从旧侧栏独立出来

- [ ] `ReviewConfirmation` 是否作为正式确认动作入口
- [ ] annotationId 在跨类型场景的唯一性规则
- [ ] 旧 `AnnotationPanel` 在 M5 结束后的过渡形态

## 四、前端 Owner 清单

- [ ] FE1：拆出独立 `AnnotationToolbar`
- [ ] FE2：拆出 `AnnotationListPanel`，承接浏览、定位、编辑入口
- [ ] FE3：收口 viewer 定位、高亮、显示对象逻辑
- [ ] FE4：明确 `ReviewConfirmation` 与确认记录区边界
- [ ] FE5：将旧 `AnnotationPanel` 降级为兼容壳层或组合容器

## 五、后端 Owner 清单

- [ ] BE1：冻结四类批注的数据结构、主键、refno/objectIds 规则

- [ ] BE3：复核确认记录接口字段与权限范围
- [ ] BE4：明确确认失败时前端可依赖的错误反馈语义

## 六、联调 / 测试清单

- [ ] INT1：联调工具栏模式切换与 viewer 工具启停一致性
- [ ] INT2：联调批注列表切换、viewer 定位、高亮、详情查看稳定性
- [ ] INT3：覆盖四类批注从列表联动到 viewer 的完整路径
- [ ] INT4：跑通“创建批注 -> 确认 -> 记录落库 -> 刷新历史”闭环
- [ ] QA1：验证工具、列表、确认动作三条链可独立使用且互相联动
- [ ] QA1-视觉基线：批注工具栏、批注列表、确认动作按 `review-reviewer.pen` 验收
- [ ] QA2：验证旧入口迁移后不误伤 M4 工作台与 M6 评论入口

## 七、风险提醒

- [ ] 风险：viewer 挂载位置和尺寸适配若不先拍板，工具栏很容易返工
- [ ] 风险：旧 `AnnotationPanel` 内逻辑分支多，拆分时容易复制状态和事件
- [ ] 风险：refno / objectId 映射不稳会直接影响定位和高亮可信度
- [ ] 风险：`clearAll()` 时机处理不好，会误伤确认与批注状态
