# 三维校审系统使用说明

> 版本：M1-M2 基线版本
> 更新时间：2026-03-15
> 适用范围：plant3d-web 三维校审功能

---

## 一、系统概述

三维校审系统提供完整的**设计编校审 → 多级审核 → 流程流转**闭环，支持：

- ✅ 设计人员发起编校审单
- ✅ 多角色审核流程（编制 → 校对 → 审核 → 批准）
- ✅ 三维模型批注与测量
- ✅ 工作流历史追踪
- ✅ 实时任务同步

---

## 二、快速开始

### 访问地址

- **前端地址**：`http://127.0.0.1:3101/`
- **后端地址**：`http://127.0.0.1:3100/`

### 角色说明

| 角色 | 职责 | 主要操作 |
|------|------|----------|
| 设计人员 | 发起编校审单 | 选择构件、填写表单、上传附件 |
| 校对人员 | 第一级审核 | 批注、测量、确认流转至审核 |
| 审核人员 | 第二级审核 | 审核确认、确认流转至批准 |
| 批准人员 | 最终审批 | 最终通过或驳回 |

---

## 三、操作流程

### 步骤 1：进入系统

1. 打开浏览器访问 `http://127.0.0.1:3101/`
2. 在项目列表中点击项目卡片（如 "AvevaMarineSample"）
3. 进入主界面，显示三维视图和工具面板

**界面说明**：
- 系统入口显示可用项目列表
- 点击项目卡片后进入主工作区
- 主界面包含：三维视图区、工具面板、菜单栏

---

### 步骤 2：切换用户角色

**操作路径**：右上角用户选择器 → 选择目标角色

**示例**：
- 切换到 "李审核员"（后端用户 `user-002`）
- 界面头部显示当前用户身份

**界面说明**：
- 右上角显示当前用户名
- 点击用户名打开角色选择下拉菜单
- 选择目标角色后，界面自动刷新并显示对应权限的功能

---

## 四、设计人员操作指南

### 4.1 发起编校审单

**操作路径**：校审菜单 → 发起编校审

**步骤**：

1. **选择构件**
   - 在三维视图中选择需要审核的构件
   - 系统自动提取构件编号（refNo）和属性

2. **填写编校审单**
   - 数据包名称：必填
   - 描述信息：选填
   - 选择处理人：
     - 校对人员（checkerId）
     - 审核人员（approverId）
   - 优先级：高/中/低
   - 截止时间：选填

3. **上传附件**（可选）
   - 支持拖拽上传
   - 附件自动关联到任务

4. **提交任务**
   - 点击"创建编校审单"按钮
   - 任务状态变为 `submitted`（待审核）

**关键字段说明**：
- `taskId`：系统生成的技术标识
- `formId`：业务标识（嵌入模式下透传）

---

### 4.2 查看任务状态

**操作路径**：校审菜单 → 我的任务

**任务状态**：
- `draft` - 草稿
- `submitted` - 待审核
- `in_review` - 审核中
- `approved` - 已通过
- `rejected` - 已驳回
- `cancelled` - 已取消

---

## 五、审核人员操作指南

### 5.1 打开审核收件箱

**操作路径**：校审菜单 → 待审核

**界面说明**：
- 显示当前用户的待审核任务列表
- 任务卡片显示：
  - 任务标题
  - 发起人
  - 构件数量
  - 当前节点
  - 任务状态

**界面示例**：
```
校对人员: user-002 | 共 1 个任务
┌─────────────────────────────┐
│ 测试编校审单-1767928364        │
│ 状态：待审核中               │
│ [处理校对]                   │
└─────────────────────────────┘
```

---

### 5.2 开始审核任务

**操作**：点击任务卡片上的 **"处理校对"** 按钮

**系统行为**：
1. 任务进入审核工作台
2. 自动加载任务上下文：
   - 任务标题
   - 模型名称
   - 发起人信息
   - 审核人员
   - 构件数量
   - 当前节点
   - 工作流历史（如有）

**工作台显示示例**：
```
当前审核任务：测试编校审单-1767928364
模型：test-model
发起人：系统用户
校对：审核人
构件数：1
当前节点：校对
```

---

### 5.3 执行批注和测量

**工具栏功能**：
- 🖱️ 选择工具
- ✏️ 画笔工具
- ⬜ 矩形工具
- ↗️ 箭头工具
- 🔤 文字工具
- 🎨 颜色选择
- ↩️ 撤销
- ✅ 保存批注

**操作步骤**：
1. 选择批注工具
2. 在三维视图中标注问题点
3. 添加文字说明
4. 点击"保存批注"确认

**数据保存**：
- 批注数据自动关联到当前任务
- 支持多次批注累积

---

### 5.4 确认流转到下一环节

**操作路径**：工作台 → **"确认流转至审核"** 按钮

**对话框内容**：
- 标题：确认提交流转
- 当前节点：校对
- 目标节点：审核
- 流转备注：选填

**步骤**：
1. 点击"确认流转至审核"
2. 填写流转备注（可选）
3. 点击"确认提交流转"
4. 任务流转到下一环节

**API 调用**：`POST /api/review/tasks/{id}/submit`

---

### 5.5 确认驳回流转到指定节点

**操作路径**：工作台 → **"确认驳回流转"** 按钮

**对话框内容**：
- 标题：确认驳回流转
- 目标节点：编制（设计）
- 流转驳回原因：**必填**

**步骤**：
1. 点击"确认驳回流转"
2. **必须填写流转驳回原因**（否则按钮禁用）
3. 点击"确认驳回流转"
4. 任务退回到指定节点

**验证规则**：
- ❌ 流转驳回原因为空时，"确认驳回流转"按钮禁用
- ✅ 填写原因后，按钮激活

**API 调用**：`POST /api/review/tasks/{id}/return`

## 六、工作流节点说明

### 节点流转路径

```
sj（编制）→ jd（校对）→ sh（审核）→ pz（批准）
    ↑          ↑          ↑          ↑
    └──────────┴──────────┴──────────┘
           可驳回到任意前置节点
```

### 节点权限

| 节点 | 中文名称 | 可执行操作 |
|------|----------|-----------|
| sj | 编制 | 确认流转至校对 |
| jd | 校对 | 确认流转至审核 / 确认驳回流转到编制 |
| sh | 审核 | 确认流转至批准 / 确认驳回流转到编制或校对 |
| pz | 批准 | 确认最终批准 / 确认驳回流转到任意节点 |

## 七、常见问题

### Q1：收件箱显示"暂无审核任务"？

**原因**：
- 当前用户没有待审核任务
- 用户角色不匹配

**解决方案**：
1. 确认已切换到正确的审核人员身份
2. 检查任务是否已分配给当前用户
3. 刷新任务列表

---

### Q2：工作流历史不显示？

**已知问题**：M2-008
- 当前版本工作流历史可能显示"暂无确认记录"
- 已在 commit `48d74a0` 修复

**临时方案**：
- 通过任务详情查看历史记录
- 等待系统更新

---

### Q3：驳回按钮无法点击？

**原因**：驳回原因未填写

**解决方案**：
- 在"流转驳回原因"文本框中输入内容
- 按钮会自动激活

---

### Q4：三维视图显示 WebGL2 错误？

**错误信息**：`需要 WebGL2（canvas.getContext("webgl2") 失败）`

**影响范围**：
- 三维模型渲染失败
- 不影响任务列表和工作流操作

**解决方案**：
1. 更新浏览器到最新版本
2. 检查显卡驱动
3. 使用支持 WebGL2 的浏览器（Chrome/Edge/Firefox）

## 八、技术说明

### API 端点

| 功能 | 方法 | 端点 |
|------|------|------|
| 创建任务 | POST | `/api/review/tasks` |
| 获取任务列表 | GET | `/api/review/tasks` |
| 获取任务详情 | GET | `/api/review/tasks/{id}` |
| 确认流转到下一环节 | POST | `/api/review/tasks/{id}/submit` |
| 确认驳回流转到指定节点 | POST | `/api/review/tasks/{id}/return` |
| 兼容直接通过/驳回 | POST | `/api/review/tasks/{id}/approve\|reject` |
| 保存确认记录 | POST | `/api/review/records` |
| 获取工作流历史 | GET | `/api/review/tasks/{id}/history` |

### WebSocket 实时通知

- **全局任务通知**：`/ws/review`
- **用户专属通知**：`/ws/review/user/{userId}`

**通知内容**：
- 任务状态变更
- 新任务分配
- 工作流流转
- 确认记录更新
