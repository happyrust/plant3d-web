## 三、网格绘制功能实现

### 3.1 CAD 网格类 (CadGrid)

**文件路径：** `src/viewer/dtx/dtxCadGrid.ts`

#### 核心实现代码

```typescript
export class CadGrid {
  private _gridSize: number;        // 网格大小
  private _minor: GridHelper | null = null;  // 细网格
  private _major: GridHelper | null = null;  // 粗网格

  constructor(options: CadGridOptions = {}) {
    // 默认网格大小为 100000（场景单位）
    this._gridSize = Math.max(1, Number(options.initialSize ?? 100000));

    // 粗细网格配置
    this._minorDivisions = clampInt(options.minorDivisions ?? 100, 10, 200);
    this._majorDivisions = clampInt(options.majorDivisions ?? 10, 2, 100);

    // 创建网格
    this._rebuild();
  }

  private _rebuild(): void {
    // major（稀疏、深色）
    this._major = new GridHelper(this._gridSize, this._majorDivisions,
                                  this._centerColor, this._majorColor);

    // minor（密集、浅色）
    this._minor = new GridHelper(this._gridSize, this._minorDivisions,
                                  this._centerColor, this._minorColor);

    this.group.add(this._major);
    this.group.add(this._minor);
  }

  // 根据包围盒调整网格大小
  fitToBoundingBox(bbox: Box3): void {
    const size = bbox.getSize(new Vector3());
    const maxXY = Math.max(size.x, size.y);
    const desired = Math.max(10, maxXY * 2);
    const nextSize = niceRoundUp(desired);

    this._gridSize = nextSize;
    this._snapStep = Math.max(1e-6, this._gridSize / this._majorDivisions);

    if (sizeChanged) {
      this._rebuild();  // 重建网格
    }
  }
}
```

## 四、关键问题：网格单位与设置单位不一致

### 4.1 问题分析

#### 网格绘制使用的单位

```typescript
// dtxCadGrid.ts 第 92 行
this._gridSize = Math.max(1, Number(options.initialSize ?? 100000));
```

- **网格默认大小：** 100000 场景单位
- **单位来源：** 硬编码的数值，没有使用单位配置

#### 场景单位的确定

```typescript
// ViewerPanel.vue 第 113 行
let scale = unitSettings.modelUnit.value === "mm" ? 0.001 : 1;
```

| modelUnit | 场景单位 | 说明 |
|-----------|----------|------|
| `'mm'` | 米（m） | mm 数据乘以 0.001 归一化到米 |
| `'m'` | 米（m） | 场景是米制 |
| `'raw'` | 原始单位 | 保持原始数据单位 |

#### 网格大小计算

```typescript
// dtxCadGrid.ts 第 111-118 行
fitToBoundingBox(bbox: Box3): void {
  const size = bbox.getSize(new Vector3());
  const maxXY = Math.max(size.x, size.y);
  const desired = Math.max(10, maxXY * 2);  // ← 使用场景单位（米）
  const nextSize = niceRoundUp(desired);
  this._gridSize = nextSize;
}
```

### 4.2 单位不一致的具体表现

| 配置项 | 场景单位 | 网格默认值 | 网格计算单位 | 问题 |
|--------|----------|------------|--------------|------|
| `modelUnit='mm'` | 米（m） | 100000 m = **100公里** | 场景米 | 默认值过大 |
| `modelUnit='m'` | 米（m） | 100000 m | 场景米 | 默认值过大 |
| `modelUnit='raw'` | 原始（可能是mm） | 100000（原始单位） | 原始单位 | 可能合适 |

### 4.3 GridHelper 本身的限制

**Three.js GridHelper 特性：**
- GridHelper 是 Three.js 的内置辅助对象
- **只绘制网格线，不显示任何文本标签**
- 没有 API 可以添加单位标签（如 "10m", "20m"）
- 需要自己实现文本标签功能

**代码证据：**
```typescript
// dtxCadGrid.ts 只使用了 GridHelper 的基础功能
this._major = new GridHelper(this._gridSize, this._majorDivisions,
                              this._centerColor, this._majorColor);
// 没有添加任何文本标签或单位显示
```

## 五、单位转换逻辑

### 5.1 单位格式化工具

**文件路径：** `src/utils/unitFormat.ts`

```typescript
const UNIT_TO_METERS: Record<LengthUnit, number> = {
  m: 1,
  cm: 0.01,
  mm: 0.001,
}

export function formatLengthMeters(
  valueMeters: number,
  unit: LengthUnit,
  precision: number,
  opts?: { suffix?: boolean }
): string {
  const v = convertLength(valueMeters, 'm', unit)
  const text = formatNumber(v, precision)
  return opts?.suffix === false ? text : `${text}${unit}`
}
```

### 6.2 存在的问题

#### 问题 1：网格无单位标签

- GridHelper 本身不显示文本
- 用户无法知道网格刻度代表什么单位
- 需要额外实现文本标签功能

#### 问题 2：默认网格大小不合理

- 硬编码为 100000 场景单位
- 在米制场景下相当于 100 公里，过大
- 首次加载模型前可能显示不合理的网格

#### 问题 3：单位系统混淆

- `modelUnit` 控制数据加载时的缩放
- `displayUnit` 控制数值显示格式
- 网格使用场景单位（受 modelUnit 影响）
- 但网格本身没有显示单位信息

## 八、建议改进方向

### 8.1 添加网格单位标签

**目标：** 在网格上显示刻度值和单位

**实现方案：**
- 在 CadGrid 类中添加文本标签渲染
- 使用 CSS2DRenderer 或 Sprite 显示刻度值
- 根据显示单位动态调整标签

**示例代码结构：**
```typescript
class CadGrid {
  private _labels: CSS2DObject[] = [];

  private _createLabels() {
    // 在主网格线上创建文本标签
    for (let i = 0; i <= this._majorDivisions; i++) {
      const position = ...;
      const value = this._gridSize * (i / this._majorDivisions);
      const text = formatLengthMeters(value, this._displayUnit, this._precision);
      const label = new CSS2DObject(text);
      label.position.copy(position);
      this.group.add(label);
    }
  }
}
```

### 8.2 优化默认网格大小

**目标：** 根据模型单位设置合理的初始值

**实现方案：**
```typescript
const DEFAULT_GRID_SIZES: Record<ModelUnit, number> = {
  'mm': 100,    // 归一化后场景是米，100米合理
  'm': 100,     // 场景是米，100米合理
  'raw': 100000, // 原始数据，保持较大值
};

constructor(options: CadGridOptions = {}) {
  const modelUnit = unitSettings.modelUnit.value;
  const defaultSize = DEFAULT_GRID_SIZES[modelUnit] ?? 100000;
  this._gridSize = Math.max(1, Number(options.initialSize ?? defaultSize));
}
```

### 8.3 统一单位系统

**目标：** 明确三个单位的概念和关系

| 单位类型 | 作用 | 影响范围 |
|----------|------|----------|
| `modelUnit` | 控制数据加载缩放 | 场景中的几何体大小 |
| `displayUnit` | 控制数值显示格式 | UI 中的文本显示 |
| **网格单位** | **显示网格刻度** | **网格标签和间距** |

**建议：**
- 在设置界面添加"网格单位"说明
- 网格单位应与 displayUnit 保持一致
- 添加网格单位切换选项
