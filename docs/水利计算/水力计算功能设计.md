# 水力计算功能设计说明

## 1. 功能入口与页面结构

- **入口位置**
  - Ribbon 菜单：`工具` → `核心` → `水力计算`
  - 点击后打开 Dock 面板：`水力计算`（可与 `测量/批注/属性` 等并列）

- **页面布局（Dock 面板内部）**
  - 顶部 Tab：
    - `快速计算`
    - `现有模型(手算)`
    - `E3D自动计算`
  - 每个 Tab 内部均采用“参数区 + 操作区 + 结果区”的结构。

## 2. 快速计算（Rapid）UI 设计

参考 `快速计算软件2  参考输入形式.jpg` 与 `初步水力计算软件-参考.py`：

- **风速参数**
  - 风管风速（m/s）
  - 设备风速（m/s）
  - 风口风速（m/s）

- **管路参数**
  - 直管长度（m）
  - 弯头数量
  - 三通数量
  - 风阀数量
  - 突变径数量

- **设备阻力（开放输入）**
  - 过滤器阻力（Pa）
  - 冷却器阻力（Pa）
  - 加热器阻力（Pa）
  - 碘吸附器阻力（Pa）
  - 末端风口阻力（Pa）
  - 其他阻力（Pa）

- **操作与输出**
  - `计算总阻力`：输出 `总阻力: xxx Pa`

> 当前实现遵循参考 Python 示例中的“示例公式”。后续若要替换为工程准确公式，建议把计算逻辑抽到独立模块（例如 `src/domain/hydraulic/calc.ts`），并记录输入/输出字段映射。

## 3. 现有模型(手算)（Manual）与三维联动

### 3.1 目标

根据 `需求描述.md`：

- 设计者从新风入口开始，沿管路逐段选择/录入，直到最末端支路。
- 每段管路由设计者输入风量、尺寸等参数，系统计算沿程/局部损失，并汇总最不利管路。

### 3.2 当前最小可用联动（已实现）

- **选择管段**按钮：
  - 点击后进入“等待选取”状态
  - 用户在 3D Viewer 中点击任意对象
  - 系统会读取当前全局 `SelectionStore.selectedRefno`，作为该管段 ID

技术实现要点：

- 3D Viewer 的点击拾取逻辑位于：`src/composables/useXeokitTools.ts`
  - 当工具模式为 `none` 时，点击模型会调用 `viewer.scene.setObjectsSelected([pickedId], true)`
  - 并调用 `syncSelectionStoreFromScene(viewer)` → `selection.loadProperties(refno)`
- 水力计算面板通过 `useSelectionStore()` 读取 `selectedRefno`，在手算 Tab 中捕获“正在选择”的那一次选中。

### 3.3 推荐的后续增强

为了让手算模式更贴近工程使用，建议后续逐步增强：

- **高亮与定位**
  - 在管段列表中提供 `定位/高亮` 按钮：
    - 定位：调用 `viewer.cameraFlight.flyTo({ aabb: viewer.scene.objects[id].aabb, ... })` 或 `flyTo({ component: id })`
    - 高亮：`viewer.scene.setObjectsSelected([id], true)`

- **管段元数据自动填充**
  - 通过 `pdmsGetUiAttr(refno)` 获取属性（当前 Properties 面板使用同一接口）。
  - 约定：从属性中提取风管尺寸、类型（弯头/三通/阀门）等字段，自动填充到手算节点。

- **拓扑输入**
  - 现阶段只做“节点列表”。后续可以升级为“有向图/管网拓扑”：
    - 节点：风口/设备/分支点
    - 边：风管段
  - 支持自动计算“最不利管路”：在拓扑上做路径枚举/最短路（最大阻力）计算。

## 4. E3D自动计算（Auto）与三维联动方案

### 4.1 目标

根据 `需求描述.md`：

- 自动识别 E3D 风管布置，尤其三通
- 自动产生管路与管道尺寸
- 设计者仅输入每段风量即可完成水力计算

### 4.2 现阶段可落地的技术路线（建议）

- **对象粒度与 ID**
  - 当前 xeokit 模型对象的 `id` 使用 `refno`（见 `src/aios-prepack-bundle-loader.ts`：`createEntity({ id: refno, ... })`）。
  - 这保证了“前端三维对象 ↔ 后端 E3D/PDMS 节点”可以直接用同一 ID 对应。

- **自动识别的关键数据**
  - 需要至少两类信息：
    - 元件类型（直管/弯头/三通/阀门/风口/设备等）
    - 连接关系（上游/下游/分支）

- **数据来源建议**
  - 后端新增 API：
    - `GET /api/hydraulic/network?root_refno=...`：返回该系统的管网拓扑（节点、边、类型、尺寸）
  - 前端只负责：
    - 指定入口/系统根节点
    - 展示拓扑
    - 输入风量
    - 触发计算

> 目前前端已有的 E3D API 主要是树形（children/ancestors/subtree-refnos），并不包含拓扑连接信息；要做“自动识别三通并生成管路”，更推荐由后端生成拓扑。

## 5. 与“模型树/属性面板”协同

- 3D 选中对象会驱动 `SelectionStore`，属性面板会自动查询并展示。
- 水力计算面板在手算模式可直接复用同一 `selectedRefno`。
- 后续若需要“从模型树选择管段并加入水力计算”，可在模型树点击时调用相同的 selection 流程（让 selection store 成为统一选择源）。
