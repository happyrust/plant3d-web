# 1. 概述

# 1.1.项目背景

核岛通风系统管网复杂、风管结构标准化程度较低，现场多采用传统的板材切割和人工组装的方式，费时、费力、费料。为提高核岛通风管道标准化、自动化、数字化和模块化水平，降低气密性风管焊接工作量，引入民用市场已经成熟的螺旋风管和BIM预制技术，通过技术改进以适应核电领域应用。但目前采用的三维布置软件E3D，缺乏可以兼容不同软件的螺旋风管元件库，需要进行开发并支持导入不同软件中使用。

螺旋风管属于特殊阻力部件，水力计算方法有待优化。参考流体力学计算公式，开发一套基于螺旋风管的水力计算软件，通过元件拖拽和相互连接搭建简化通风模型，支持导入螺旋风管阻力特性和风量值，快速输出水力计算结果。软件适用于传统的定风量系统，也能通过调节平衡阀开度、孔板规格或采用变风量阀、风口，适用于变风量系统。

负荷计算确定的风量是水力计算的输入参数，现有负荷计算多采用稳态传热学公式，对于服务房间多的大系统，计算效率较低，尤其对于非稳态用户适用性较差。需要研发一套专用负荷计算软件，将传统计算方法封装成计算界面，将通风房间建立成中国核电工程有限公司

固定元件，并能选用风量调节装置实现变风量功能，通过适当调整输入参数快速完成系统负荷计算。

# 1.2.适用范围

本外委任务书适用于中国核电工程有限公司自主研发项目 KY24085 课题中“螺旋风管元件库及设计软件开发”的相关工作。任务承担单位（以下简称乙方）应按要求完成相关开发工作。软件开发除满足本文件的要求外，还应满足国家相关法规、标准、行业标准规范的规定，符合中国核电工程有限公司（以下简称甲方或 CNPE）及其上级主管单位（中国核工业集团公司）发布的与科研工作有关的文件、规定、工作程序的要求。

# 1.3.委托内容

针对核电站用螺旋风管，开发一套可兼容不同设计软件的螺旋风管元件库，以及基于螺旋风管系统模型的水力和负荷计算专用软件。元件库主要用于E3D软件中对通风系统进行三维建模三维设计，并可对接建立和CAMduct软件的接口，实现自动排版、下料等功能。负荷、水力计算软件，用于螺旋风管系统及其他通风系统的设计计算，

采用数字化工具提升计算效率。具体开发内容包括：

# (1) 开发螺旋风管元件库

中国核电工程有限公司

针对三维布置软件（E3D），建立可调用的螺旋风管元件库，可实现 UDA 建模功能。

完成的三维模型导出后也需要适配于Navisworks软件进行模型的展示。

提供适配有限元分析的模型，适配于主流仿真软件ANSYS、FLUENT、ABAQUS、FLOWMASTER等。

对建好的整个通风模型支持导入 CAMduct 软件，可以实现将三维管件螺旋风管展开二维部件进行二维 UV 展开，自动在标准钢板上进行最优化排版。元件库应具备用户自行维护的功能，确保设计参数的开放性。

# (2) 开发负荷、水力计算专用软件

基于已有成熟水力、负荷计算方法，封装计算界面，研发螺旋风管系统模型的水力和负荷计算专用软件，适用于多种系统形式，如全新风系统、一次回风系统，可实现变风量功能。通过元件拖拽和相互连接搭建通风系统，仅需适当调整参数，即可快速输出负荷、水力计算书。

# 3.技术要求

# 3.1.总体要求

软件应满足甲方所需功能要求，界面简洁，交互友好，计算速度快，计算精度满中国核电工程有限公司

足工程要求，运行稳定可靠。需向甲方提供全部源代码，并提供不少于3年的售后和维护服务。

# 3.1.1 运行环境要求

操作系统：要求能够支持目前通用的主流操作系统环境，包括win7、win8、win10等，包括32位和64位系统，建议同时支持Linux。

# 3.1.2性能要求

在某种运行环境下，系统应达到的响应时间、处理时间、处理量等要求。如：

响应时间：一般情况下要求<1s，使用高峰期情况下要求<3s；

吞吐量：150000条/日的数据量；

数据转换和传递时间：模块间的数据交换，要求时间不超过2秒

并发：同一时间下，同一功能模块的并发用户数：100。

# 3.1.3安全要求

系统安全要求参照国家信息系统等保三级标准执行，达到等保二级测评要求。系统应提供完善的安全防范功能，包括数据存储加密、未授权用户限制、授权用户权限控制等，系统操作审计等要求，系统备份要求等。

(1) 对于系统中的敏感数据的传输和存储必须使用加密算法进行加密，以保证信息的安全性；  
(2) 需对输入应用系统数据的完整性及合理性加以验证;  
(3) 需提供完整的系统用户操作日志功能。  
(4) 数据库采用热备方案，重要数据文件的存取要有安全措施保障，保证数据的安全性，将系统的恢复时间控制在小时级。

# 3.1.4其他要求

# (1) 可靠性

系统开发测试过程中，根据相关要求将对系统功能进行全过程、全业务的测试工作，确保单元测试、集成测试等环节对测试案例的覆盖率达  $100\%$ 。

在承受最大并发用户数持续运行2小时的情况下，系统运行平稳，业务失败率不超

过  $0.1\%$  ，内存占用率没有明显增长且1小时后内存恢复初始值。

在承受百分之四十的最大并发用户数持续运行8小时的情况下，系统运行平稳，业务失败率不超过  $0.1\%$  ，内存占用率没有明显增长且1小时后内存恢复初始值。

# (2) 可维护性

本次系统建设将通过从可理解性、易扩展性和易测试性等三方面提高系统的可维护性。包括以下各方面：

每段代码都必须具有完整的、易理解的注释信息。

每段代码的输入输出必须明确成功执行的标准。

具有清晰、易懂的、完备的设计和使用文档，并对其内容是否符合要求具有明确的判定标准。

开发软件具有完备、规范的应用日志及管理功能；包括：运行日志、应用日志、批处理日志、文件传输日志、数据库操作日志、数据修改、维护日志等等。

一般要求代码的循环复杂度不超过7；所有对象方法的代码尽量不超过200行。

一个模块的最大循环复杂度不能超过20层。

# 3.5.结构化输出文档

软件计算结果可输出为计算书，采用结构化文档形式，可参考商用结构分析软件输出计算书。

# 3.6.外部接口要求

螺旋风管元件库可导出多种文件格式，包括但不限于stl、obj、step、step、maj、gltf(JSON)等。

(1) 三维布置平台接口

元件库可导入CNPE内部三维布置平台设计软件。

(2) 结构分析软件接口

CNPE 现采用 RFEM 软件进行有限元结构分析，软件首先可将需要有限元计算的螺旋风管模型转化为 RFEM 可读的 Excel 数据表输出；计算后 RFEM 生成输出数据表，再经软件读入转化为工程设计选型结果。

软件必须在满足现有业务量需求的基础上，对今后的业务发展进行有效的评估，使系统处理能力在一定的时间内能够满足业务增长带来的处理能力增长的需求。

服务商原则应使用本公司正式员工，不允许使用外包人员。项目人员至少具备大中国核电工程有限公司

学本科学历。服务商提供项目开发人员信息，包括劳动关系证明材料，专业资质，学历，从事开发工作的时间，工作经历，业绩等。

# 7.知识产权

本项目执行过程中所产生的成果、专利所有权由甲乙双方协定，未经书面许可，不得以任何方式复制、传播、发表和外传，项目在执行中要保护知识产权，防止侵权；具体包括如下要求：

(1) 乙方应严格按照科研外委合同履行知识产权权属、使用、保密等约定;  
(2) 工作范围内产生的知识产权和成果未经甲方许可，乙方不得单独公开发表、使用或转让；  
(3) 工作范围内产生的软件著作权由甲方提出申请;  
(4) 工作范围内产生的成果由甲方提出鉴定和申报。

# 8. 验证和验收

# 8.1. 验证

实施方在开发完成后应编制测试计划，并据此展开内部测试。

内部测试完成后，提交测试版本。根据用户反馈的意见进行修改。

修改完成后，由公司组织评审。实施方根据评审结论完成正式版，并交付。

# 8.2. 验收

只有项目交付成果正式提交中核工程后，服务商才能向中核工程提出（阶段性）验收申请。

项目完成后，服务商须提前十四天以书面方式，向中核工程提出验收申请。

采用评审验收方式，包括方案评审、最终成果评审。方案评审及最终成果评审由甲方参与，乙方承办会议并负责评审产生的所有费用。

项目结束后，实施方在质保期内按合同规定提供技术支持，对程序进行维护。

# 9. 其它

# 9.1.后续服务

乙方应保留过程程序、文件、数据、草稿、资料等，配合甲方回答相关专家单位（若有）的审查。乙方应提供不少于3年的售后和维护服务。

终验后的现场技术支持服务解决在系统上线后出现的超出本项目范围的需求，且为小规模、小量级、与本期系统上线应用比较相关的需求，由于需求不确定，一般按工作量结算。可根据项目大小，设置1-6个人月即可。

质保服务主要解决本项目交付物在项目质保期内出现的系统原生问题。包括补丁升级、软件疑难问题处理等软件维护服务，服务内容主要为以下范围：

(1) 及时响应并解决软件使用过程中产生的问题，确保软件正常使用。  
(2) 小规模的功能调整改进。

平台最终验收通过后，实施方应提供3年技术支持服务，保证至少有2名工程师具备及时到达委托方现场，响应委托方紧急需求的能力，解决质保服务之外的问题。

实施方应向任务委托单位提交的服务如下：

现场技术支持服务  
培训服务  
— 质保服务
