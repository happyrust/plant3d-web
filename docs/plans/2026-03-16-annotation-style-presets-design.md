# 批注样式预设卡片设计

## 目标

把当前 `批注样式` 面板里的三组快捷预设从普通按钮升级成更可读的主题卡片，让用户在不切回场景的情况下，直接理解 `柔和 / 清晰 / 强强调` 的视觉取向。

## 现状

- 当前面板已经有：
  - 四类批注引线参数编辑
  - 实时预览条
  - 三个快捷预设按钮
- 当前问题是：
  - 预设按钮缺少语义说明
  - 用户只能靠名称猜效果
  - “预设”和“手动参数”之间的层级关系不够清晰

## 方案对比

### 方案 A：普通按钮 + 说明文本

- 在按钮下方补一句描述
- 改动最小
- 但视觉层次仍然弱，预设不够像“可选主题”

### 方案 B：可视化预设卡片

- 每个预设显示：
  - 名称
  - 一句短描述
  - 一条缩略引线预览
- 点击整张卡片即可套用预设
- 这是本次推荐方案

### 方案 C：下拉选择器

- 用一个 select 管理三套预设
- 节省空间
- 但可发现性和可比较性都更差

## 采用方案

采用方案 B。

## 交互规则

- 面板顶部显示三个预设卡片：
  - `柔和`：更轻、更淡，适合默认浏览
  - `清晰`：当前默认值，平衡清楚与克制
  - `强强调`：更粗、更亮，适合远距离查看和重点标注
- 点击卡片后：
  - 调用 `useAnnotationStyleStore().applyPreset()`
  - 立即更新四类批注样式
  - 预览条同步刷新
- 当前不新增“当前选中预设”的强绑定状态，只做即时套用；后续如果需要，再补“匹配当前配置”的高亮逻辑

## 实现范围

- 修改 `src/components/tools/AnnotationStylePanel.vue`
  - 把顶部快捷按钮替换成卡片式入口
  - 增加说明文案与简化预览
- 保持 `src/composables/useAnnotationStyleStore.ts` 的预设接口不变
- 补充/更新组件测试，确保：
  - 卡片存在
  - 点击卡片会套用预设
  - 卡片内的说明文本可见

## 非目标

- 不新增后端或持久化协议
- 不新增复杂的“当前配置匹配哪个预设”的自动推导
- 不调整场景里的引线渲染实现
