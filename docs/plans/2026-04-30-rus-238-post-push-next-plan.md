## 3. 阶段计划

### Phase 1 · 验收输入收集

需要输入：

- 目标 BRAN。
- PMS 包名或任务单。
- 测量样例：距离、角度、带批注证据的确认记录。
- 验收入口：本地三维、仿 PMS、真实 PMS 嵌入页。
- 验收角色：至少 SJ 与 JH。

通过条件：

- 输入记录到 `2026-04-30-rus-238-measurement-path-acceptance-plan.md`。
- 可以复现同一测量在列表、批注证据、确认回放三个入口的展示。

### Phase 2 · 本地真实模型验收

执行内容：

- 加载目标模型。
- 新建距离测量和角度测量。
- 检查三类展示状态：
  - 初始 refno fallback。
  - lookup resolved 后完整路径。
  - lookup 失败或空数据 fallback。
- 执行定位、隐藏、恢复、删除。

通过条件：

- UI 不出现 `o:...:0`。
- 完整路径可读且不会导致列表布局不可用。
- 操作行为无退化。

### Phase 3 · 批注与确认回放验收

执行内容：

- 给批注关联距离和角度测量。
- 检查 `AnnotationWorkspace` 中关联测量证据展示。
- 生成确认记录后打开 `TaskReviewDetail`。
- 对比同一测量点在三处入口的文案。

通过条件：

- 批注证据和确认回放与测量列表展示一致。
- 批注定位按钮仍按原有 `engine/id` 工作。
- 历史确认记录 fallback 正常。

### Phase 4 · PMS/编校审验收

执行内容：

- 从 PMS 或仿 PMS 入口进入三维校审。
- 使用目标 BRAN 完成测量、批注证据、确认记录。
- 切换校核入口或重新打开任务。
- 记录包名、BRAN、角色、截图和实际展示文本。

通过条件：

- PMS/编校审流程不暴露内部 object id。
- lookup 成功时展示完整路径。
- lookup 失败时仍展示规范化 refno。

### Phase 5 · 是否需要二次开发

根据验收结果判断：

- 如果路径过长影响可读性：把完整路径改为 tooltip 或次级文案，主文案保留 refno / leaf name。
- 如果请求过多：给 `measurementPathLookup` 增加并发限制或批量 lookup。
- 如果批注证据缺失原始测量记录：补展示层字段，不把同步模型改成异步。
- 如果 PMS 场景路径无法 resolved：记录数据源限制，确认 fallback 是否可交付。

### Phase 6 · 工作区收敛

当前工作区仍有大量 RUS-238 之外的未提交/未跟踪变更。建议单独处理：

- 先按主题列出剩余变更来源。
- 将 PMS、批注、空间查询、配置类变更拆成独立提交或独立分支。
- 删除临时调试产物前先确认是否属于用户保留文件。
- 不把这些变更混入 RUS-238 后续补丁。
